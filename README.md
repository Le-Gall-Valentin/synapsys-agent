# SynapSys Agent

> Lightweight Go agent for executing routines on behalf of the SynapSys API.

The SynapSys agent is a single static binary deployed on each target Linux server. It connects to the [SynapSys API](https://github.com/Le-Gall-Valentin/synapsys-api) via a mutually authenticated WebSocket (mTLS), receives execution orders, runs shell scripts or Makefile targets, and streams back logs in real time.

---

## How it works

1. On first launch, the agent generates a local RSA key pair and sends a Certificate Signing Request (CSR) to the API
2. An administrator reviews and approves the enrollment request in the UI (fingerprint verification)
3. The API signs the certificate via its internal CA and returns it to the agent
4. From that point on, the agent authenticates via mTLS on every reconnection
5. If the connection drops, the agent halts — it never operates disconnected

---

## Prerequisites

- Linux (amd64 or arm64)
- A running [SynapSys API](https://github.com/Le-Gall-Valentin/synapsys-api) instance
- A dedicated Linux user with only the permissions needed to run your scripts

## Installation

### Download the binary

```bash
curl -LO https://github.com/Le-Gall-Valentin/synapsys-agent/releases/latest/download/synapsys-agent-linux-amd64
chmod +x synapsys-agent-linux-amd64
sudo mv synapsys-agent-linux-amd64 /usr/local/bin/synapsys-agent
```

Verify the checksum:

```bash
curl -LO https://github.com/Le-Gall-Valentin/synapsys-agent/releases/latest/download/checksums.txt
sha256sum --check checksums.txt
```

### First run (enrollment)

```bash
synapsys-agent \
  --api-url wss://your-api.example.com \
  --enrollment-token YOUR_ENROLLMENT_TOKEN \
  --cert-dir /etc/synapsys-agent
```

The enrollment token is generated by an `ADMIN` in the SynapSys UI. Once enrollment is complete, subsequent runs only require `--api-url` and `--cert-dir`.

### Run as a systemd service

```bash
sudo synapsys-agent install
sudo systemctl enable --now synapsys-agent
```

## Configuration

| Flag | Description |
|---|---|
| `--api-url` | WebSocket URL of the API (`wss://...`) |
| `--enrollment-token` | Enrollment token (first launch only) |
| `--cert-dir` | Directory for storing keys and certificates |
| `--log-level` | Log level: `debug`, `info`, `warn`, `error` (default: `info`) |

---

## Automatic updates

The agent checks for new versions on startup and periodically at runtime. When a new version is available:

1. The agent downloads the new binary from GitHub Releases over the existing mTLS channel
2. Verifies the GPG signature against the embedded public key
3. Replaces the binary on disk
4. Restarts via systemd

This behavior can be disabled with `--auto-update=false`.

---

## Security

- The agent runs as a **dedicated unprivileged user** — configure it with only the rights needed for your scripts
- All communication with the API is **mTLS** — both sides authenticate with certificates
- Auto-update binaries are **GPG-signed** — the agent refuses unsigned or untrusted updates
- The agent **stops immediately** if the connection to the API is lost

## Building from source

```bash
git clone https://github.com/Le-Gall-Valentin/synapsys-agent.git
cd synapsys-agent
go build -o synapsys-agent ./cmd/agent
```

Cross-compilation:

```bash
GOOS=linux GOARCH=amd64 go build -o synapsys-agent-linux-amd64 ./cmd/agent
GOOS=linux GOARCH=arm64 go build -o synapsys-agent-linux-arm64 ./cmd/agent
```
